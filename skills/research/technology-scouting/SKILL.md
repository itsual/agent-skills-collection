# Technology Scouting

## Overview

Technology scouting systematically finds and evaluates external technologies and capabilities that could create advantage — or disrupt the current approach.

## When to Use

- Build vs buy vs partner decisions
- Innovation and R&D pipeline input
- Vendor and startup landscape scans
- Early signal detection on emerging tech

## Core Practices

- Define the capability need and constraints first
- Scan across academia, startups, vendors, and open source
- Assess maturity (research, pilot, production) honestly
- Evaluate fit: performance, integration, risk, ecosystem
- Separate hype cycles from demonstrated results
- Recommend watch / pilot / adopt / ignore with rationale

## Principles

- Scouting without a decision frame produces tourism
- Demos ≠ production readiness
- IP, talent, and ecosystem matter as much as algorithms
- Revisit landscapes; they move quickly

## Verification

- [ ] Need and evaluation criteria were defined up front
- [ ] Maturity assessments are evidence-based
- [ ] Recommendations are actionable (watch/pilot/adopt)
