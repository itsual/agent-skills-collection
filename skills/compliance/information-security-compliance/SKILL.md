# Information Security Compliance

## Overview

Security compliance demonstrates that protective controls meet external and internal requirements. It bridges security engineering and regulatory evidence.

## When to Use

- Mapping controls to ISO, SOC 2, regulatory IT requirements
- Access control and logging compliance reviews
- Secure SDLC evidence for exams
- Vendor security questionnaire responses

## Core Practices

- Map requirements to specific technical and process controls
- Enforce least privilege and periodic access reviews
- Ensure audit logs exist for sensitive actions
- Embed security checkpoints in change and delivery processes
- Test incident detection and response readiness
- Collect evidence continuously, not only at audit time

## Principles

- Checkbox security without operating effectiveness fails exams and attackers
- Evidence freshness matters
- Compensating controls must be real and monitored
- Security and compliance teams need a shared control language

## Verification

- [ ] Key requirements trace to implemented controls
- [ ] Access reviews and logging are operating
- [ ] Evidence can be produced for priority controls quickly
