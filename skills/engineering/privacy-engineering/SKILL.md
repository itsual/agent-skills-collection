# Privacy Engineering

## Overview

Privacy is a system property. It must be designed in, not bolted on after collection.

## When to Use

- Collecting, storing, or processing personal data
- Designing user profiles, analytics, or personalization features
- Responding to privacy regulations or data subject requests

## Core Practices

- Data minimization — collect only what you need
- Purpose limitation — use data only for stated purposes
- Strong access control and audit logging for personal data
- Encryption in transit and at rest
- Retention limits and deletion capabilities
- Support for data subject rights (access, deletion, portability) where required
- Privacy impact assessment for higher-risk processing

## Principles

- Privacy by design and by default
- Make data flows and retention explicit
- Treat personal data as a liability as well as an asset

## Verification

- Personal data stores and flows are documented
- Deletion and access request processes work
- Access to personal data is controlled and auditable
