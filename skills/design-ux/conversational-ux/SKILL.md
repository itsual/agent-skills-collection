# Conversational UX

## Overview

Conversational UX blends dialogue design with interface design. Users need to know what the system can do, how to steer it, and when to switch to traditional UI.

## When to Use

- AI assistants and copilots
- Chat-based support or onboarding
- Voice + chat hybrid experiences
- Guided troubleshooting dialogues

## Core Practices

- Set expectations about capability and limits early
- Design for turn-taking, repair, and clarification
- Offer quick replies and structured inputs when free text is burdensome
- Provide escape hatches to human help or classic UI
- Handle failure gracefully when the system doesn’t understand
- Log and review conversations to improve flows

## Principles

- Conversation should reduce effort, not add theater
- Be honest about uncertainty and limitations
- Let users correct and redirect easily
- Combine conversation with GUI where precision matters (forms, tables, confirmations)

## Verification

- [ ] Users understand what they can ask or do
- [ ] Repair and clarification paths exist
- [ ] Critical tasks have reliable completion paths
