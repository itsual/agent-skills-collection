# Content Design

## Overview

Content design treats language and information architecture as design materials. It’s broader than microcopy: it includes structure, hierarchy, and how content supports tasks end-to-end.

## When to Use

- Designing content-heavy product areas
- Structuring help, onboarding, or guided flows
- Aligning content strategy with product UX
- Improving comprehension in complex domains

## Core Practices

- Start from user questions and jobs
- Structure content for scanning and progressive disclosure
- Define consistent terminology and voice
- Pair content decisions with UI patterns
- Measure whether content helps task completion

## Principles

- Content is the interface for many tasks
- Front-load meaning; cut decoration words
- Consistency of terms reduces cognitive load
- Design content for the medium (UI, email, docs) deliberately

## Verification

- [ ] Users can complete tasks using the content provided
- [ ] Terminology is consistent
- [ ] Structure supports scanning and progressive detail
