# Onboarding UX

## Overview

Onboarding UX bridges signup and habitual use. It should prioritize activation — the moment users experience real value — over exhaustive education.

## When to Use

- First-run experiences for new users
- Feature adoption tours and checklists
- Activation-focused empty states
- Re-onboarding after major product changes

## Core Practices

- Define the activation moment and design backward from it
- Progressive disclosure — don’t dump every feature at once
- Combine UI guidance with optional deeper help
- Allow skip/dismiss; respect returning users
- Personalize by role or use case when data allows
- Measure completion and impact on retention

## Principles

- Time-to-value beats completeness of the tour
- Show, don’t only tell
- One successful outcome > many tooltips
- Onboarding should fade as competence grows

## Verification

- [ ] New users can reach value without external help
- [ ] Guidance is dismissible and not blocking
- [ ] Activation metrics improve when onboarding changes ship
