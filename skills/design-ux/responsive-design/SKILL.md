# Responsive Design

## Overview

Responsive design ensures a product works across a range of viewports. It’s not just stretching layouts — it’s rethinking hierarchy, navigation, and interaction for each context.

## When to Use

- Designing products used on desktop, tablet, and mobile
- Defining breakpoint strategy and layout adaptation rules
- Improving experiences that currently degrade on smaller screens

## Core Practices

- Design from content and priority outward
- Define meaningful breakpoints based on content, not only devices
- Adapt navigation, density, and interaction patterns per viewport
- Test real content at extreme widths
- Specify behavior for fluid vs fixed elements

## Principles

- Mobile is not a shrunk desktop
- Prioritize what matters most as space shrinks
- Maintain touch-friendly targets on coarse pointers
- Performance matters more on constrained devices

## Verification

- [ ] Critical tasks work at major breakpoints
- [ ] Layout does not break with real content lengths
- [ ] Navigation and primary actions remain accessible
