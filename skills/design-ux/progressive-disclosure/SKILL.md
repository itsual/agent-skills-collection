# Progressive Disclosure

## Overview

Progressive disclosure reveals complexity gradually. It keeps primary paths simple while still serving experts who need depth.

## When to Use

- Advanced settings and configuration
- Feature-rich tools with beginner and expert users
- Onboarding that shouldn’t overwhelm
- Dense forms or dashboards

## Core Practices

- Identify the core path vs optional/advanced paths
- Use clear entry points to advanced detail (“Advanced”, “More options”)
- Remember user choices when they expand complexity
- Don’t hide critical information behind disclosure
- Combine with smart defaults so most users never need the advanced layer

## Principles

- Simple by default, powerful when needed
- Disclosure controls must be obvious and reversible
- Never use progressive disclosure to bury legal or safety-critical info
- Test that experts can still work efficiently

## Verification

- [ ] New users aren’t overwhelmed by advanced options
- [ ] Experts can access depth without friction
- [ ] Critical content isn’t inappropriately hidden
