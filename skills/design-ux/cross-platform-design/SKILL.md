# Cross-Platform Design

## Overview

Cross-platform design balances consistency of brand and mental model with respect for each platform’s conventions, capabilities, and constraints.

## When to Use

- Products that span web, iOS, Android, desktop, or other surfaces
- Establishing shared patterns without forcing identical UI everywhere
- Prioritizing platform-specific investments

## Core Practices

- Define what must be consistent (identity, core concepts, key tasks) vs what should adapt
- Respect platform UI guidelines where users have strong expectations
- Share design tokens and foundations; allow component adaptation
- Design feature parity intentionally — not everything needs to ship everywhere at once
- Test on each major platform, not only the primary one

## Principles

- Consistency of meaning > consistency of pixels
- Don’t ship a lowest-common-denominator experience
- Platform differences should feel intentional, not accidental
- Shared design systems need platform-specific expression layers

## Verification

- [ ] Core tasks feel coherent across platforms
- [ ] Platform conventions are followed where expected
- [ ] Differences are purposeful and documented
