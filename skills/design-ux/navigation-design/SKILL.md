# Navigation Design

## Overview

Navigation is how users build a mental model of the product. Clear navigation reduces anxiety and supports efficient task switching.

## When to Use

- Designing or redesigning global/local navigation
- Scaling IA as features grow
- Improving wayfinding and “lostness”
- Adapting navigation across breakpoints

## Core Practices

- Reflect user mental models in structure and labels
- Keep top-level items limited and meaningful
- Show current location clearly
- Support both browse and search paths
- Design responsive navigation patterns deliberately (tabs, drawers, bottom nav, etc.)
- Test findability with tree tests or usability tasks

## Principles

- Navigation is not a sitemap dump — prioritize
- Labels beat icons-alone for clarity in most cases
- Consistency of placement builds muscle memory
- Don’t hide critical destinations behind overflow without strong reason

## Verification

- [ ] Users can predict where to find key destinations
- [ ] Current location is always clear
- [ ] Navigation scales without collapsing into chaos
