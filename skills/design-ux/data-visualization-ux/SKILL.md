# Data Visualization UX

## Overview

Data visualization UX is about helping people understand and act on information — not decorating numbers. Clarity, honesty, and hierarchy matter more than chart variety.

## When to Use

- Product analytics and admin dashboards
- Reporting and operational monitoring UIs
- Embedding charts in workflows
- Explaining trends, comparisons, and distributions

## Core Practices

- Start from the question the user needs answered
- Choose chart types that match the relationship (trend, comparison, part-to-whole, distribution)
- Design for scanning: clear titles, direct labels, restrained color
- Provide context (comparisons, time ranges, definitions)
- Support drill-down and export when the job requires it
- Handle empty, loading, and sparse-data states

## Principles

- Don’t distort; be careful with truncated axes and 3D effects
- Color should encode meaning, not just aesthetics
- Accessibility: don’t rely on color alone
- Progressive disclosure for complexity

## Verification

- [ ] A user can answer the primary question in seconds
- [ ] Chart choice matches the data relationship
- [ ] Definitions and filters are understandable
