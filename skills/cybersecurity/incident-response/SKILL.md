# Incident Response

## Overview

Incident response limits damage when prevention fails. Prepared roles, playbooks, and practice determine whether minutes or weeks decide the outcome.

## When to Use

- Building IR plans and playbooks
- Active security incidents (malware, intrusion, account takeover, data theft)
- Tabletop exercises and readiness tests
- Post-incident review and improvement

## Core Practices

- Define severity levels, roles, and escalation paths before incidents
- Triage: scope, impact, and ongoing attacker activity
- Contain to stop the bleeding without destroying evidence needlessly
- Eradicate root access paths; recover cleanly
- Communicate to stakeholders and regulators as required
- Run blameless post-incident reviews with tracked fixes

## Principles

- Panic without process increases damage
- Evidence preservation enables learning and legal needs
- Premature “all clear” is a common failure mode
- Practice under calm conditions builds muscle memory

## Verification

- [ ] Roles and severity definitions exist
- [ ] Playbooks cover top scenarios
- [ ] Tabletop or live exercises occur on a cadence
