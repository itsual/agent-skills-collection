# Security Fundamentals

## Overview

Security fundamentals are the durable principles behind controls: protect the right things, assume failure, limit blast radius, and verify. Tools change; principles persist.

## When to Use

- Security awareness for builders and operators
- Design reviews at a principles level
- Establishing a baseline control vocabulary
- Prioritizing investments when everything feels urgent

## Core Practices

- Map assets and data sensitivity before choosing controls
- Apply least privilege and need-to-know by default
- Use defense in depth — no single control as the only barrier
- Prefer secure defaults and fail-safe behavior
- Separate duties for critical sensitive actions
- Treat security as continuous risk management, not a one-time project

## Principles

- CIA triad is necessary but incomplete without accountability and authenticity
- Complexity is an enemy of security
- Perfect security is impossible — prioritize by impact and likelihood
- Usable security beats theoretical security people bypass

## Verification

- [ ] Critical assets and data classes are identified
- [ ] Least privilege is the default posture
- [ ] Controls are layered for high-impact risks
