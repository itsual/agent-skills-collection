# Endpoint Security

## Overview

Endpoints are where users work and where attackers often land. Hardening, rapid detection, and controlled privileges reduce compromise likelihood and impact.

## When to Use

- Corporate device and server hardening baselines
- EDR deployment and response workflows
- Admin rights reduction programs
- Ransomware resilience on endpoints

## Core Practices

- Maintain an inventory of endpoints and ownership
- Apply secure configuration baselines and continuous assessment
- Keep OS and critical apps patched on defined SLAs
- Deploy EDR/antivirus with monitoring and response paths
- Reduce local admin rights; use elevation workflows
- Encrypt disks and protect against physical loss

## Principles

- Unmanaged endpoints are unmanageable risk
- Detection without response capacity is incomplete
- Users will bypass painful controls — design for usability
- Backups and recovery are part of endpoint ransomware defense

## Verification

- [ ] Inventory coverage is high for corporate endpoints
- [ ] Patch and EDR coverage meet policy targets
- [ ] Admin rights are justified and reviewed
