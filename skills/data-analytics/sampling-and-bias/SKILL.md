# Sampling and Bias

## Overview

Every dataset is a sample from a process. Bias enters through who/what is included, what is measured, and what is missing. Good analysts name these limits.

## When to Use

- Survey and study design
- Interpreting observational product data
- Critiquing third-party datasets
- Generalizing results to broader populations

## Core Practices

- Define the target population vs the sampled population
- Identify selection mechanisms (who appears in the data)
- Watch for survivorship and censoring
- Quantify missing data patterns when possible
- Use weighting or design-based methods when appropriate
- State external validity limits clearly

## Principles

- Bigger biased samples are still biased
- Convenience samples need humility in claims
- Measurement bias can dominate sampling bias
- Product logs reflect product users, not all humans

## Verification

- [ ] Target vs sample population is explicit
- [ ] Major bias risks are listed
- [ ] Generalization claims are bounded
