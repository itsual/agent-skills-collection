# Metadata Management

## Overview

Metadata is data about data: schemas, owners, meanings, SLAs, and run history. Managed metadata powers discovery, governance, and automation.

## When to Use

- Building or integrating catalog, lineage, and quality platforms
- Improving documentation coverage
- Enabling automated governance controls
- Standardizing naming and classification (e.g. PII tags)

## Core Practices

- Distinguish technical, business, and operational metadata
- Automate collection from platforms wherever possible
- Enforce classification tags for sensitive data
- Standardize naming conventions for key assets
- Keep ownership and description fields mandatory for critical assets
- Use metadata to drive access policies and quality checks

## Principles

- Manual metadata rots; automation sustains
- Metadata quality has its own quality problem — manage it
- Classification accuracy matters for security and privacy controls
- Metadata should serve workflows, not only reporting on documentation %

## Verification

- [ ] Critical assets have required metadata fields populated
- [ ] Sensitive data classification is applied
- [ ] Metadata feeds catalog/lineage/quality use cases
