# Data Visualization

## Overview

Visualization is analysis and communication. Good charts reveal patterns; bad charts distort them. Choose form based on the question and the data.

## When to Use

- Exploratory plotting during analysis
- Stakeholder-facing charts and dashboards
- Comparing magnitudes, distributions, compositions, relationships
- Avoiding misleading visual encodings

## Core Practices

- Match chart type to data relationship (trend, comparison, distribution, part-to-whole)
- Label axes, units, and sources
- Start baselines appropriately (be careful with truncated axes)
- Reduce non-data ink; highlight the signal
- Design for the audience’s decision needs
- Show uncertainty when it matters

## Principles

- Dual axes often mislead
- 3D and decorative charts usually hurt comprehension
- Color should encode meaning, with accessibility in mind
- One chart, one primary message for executives

## Verification

- [ ] Chart type fits the question
- [ ] Scales and labels are honest and complete
- [ ] A naive reader can take the intended message
