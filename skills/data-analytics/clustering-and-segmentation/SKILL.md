# Clustering and Segmentation

## Overview

Segmentation groups similar entities so strategy can differentiate. Clusters must be stable enough and meaningful enough to act on — not only algorithmically neat.

## When to Use

- Customer or user segmentation
- Behavioral clustering for product or marketing
- Grouping entities for operational treatment
- Exploring structure in high-dimensional data

## Core Practices

- Select features aligned to the segmentation purpose
- Scale and preprocess appropriately
- Try multiple algorithms/k and validate (stability, separation, domain sense)
- Profile segments in human-readable terms
- Test whether segments predict different outcomes or responses
- Name and document segments for operational use

## Principles

- Segments exist to change decisions
- Unstable clusters are dangerous to operationalize
- More segments are not always better
- Domain review is part of validation

## Verification

- [ ] Purpose of segmentation is explicit
- [ ] Segments are interpretable and reasonably stable
- [ ] Action implications are defined
