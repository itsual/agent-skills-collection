# CS Tooling

## Overview

CS tools amplify process; they cannot replace it. Good tooling centralizes health, workflows, and customer context so CSMs spend time on judgment, not hunting data.

## When to Use

- Selecting a CS platform
- Integrating product usage data into health views
- Automating playbook triggers
- Improving CRM–CS system consistency

## Core Practices

- Start from motions and data needs, not feature checklists
- Integrate usage, support, and commercial data into one account view
- Automate triggers for common plays; keep overrides possible
- Enforce minimum CRM/CS hygiene standards
- Train CSMs on workflows, not only navigation
- Review stack ROI: time saved, signal quality, outcome lift

## Principles

- Tool sprawl creates contradictory truths
- Bad data in beautiful dashboards still misleads
- Adoption of the tool by CSMs is part of success
- Process design precedes automation

## Verification

- [ ] Core motions are supported end-to-end in tooling
- [ ] Account data sources are trusted and reconciled
- [ ] CSM adoption and hygiene are measured
