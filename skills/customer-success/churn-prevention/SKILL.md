# Churn Prevention

## Overview

Churn prevention combines early detection with meaningful intervention. The goal is not to “save at any cost,” but to address real value gaps or exit cleanly when fit is gone.

## When to Use

- Designing save plays and risk workflows
- Acting on health-score or usage declines
- Post-cancellation analysis and learning
- Executive escalation for strategic accounts

## Core Practices

- Define risk triggers and response SLAs
- Diagnose why risk appeared (product, service, value, politics, budget)
- Run save plays matched to root cause
- Multi-thread and escalate when strategic value warrants it
- Know walk-away criteria — not every account should be saved on bad terms
- Analyze churn reasons and feed product/CS improvements

## Principles

- Late discounts don’t fix non-adoption
- Root cause > generic “check-in” outreach
- Win-back starts with honest exit interviews
- Pattern-level churn is a product problem, not only a CS problem

- [ ] Risk triggers and plays are defined

- [ ] Churn reasons inform roadmap and process changes
