# Customer Feedback Loops

## Overview

Feedback loops complete the cycle: capture → analyze → decide → act → tell the customer. Without the last steps, feedback programs train customers that input disappears.

## When to Use

- Feature request management
- Post-release communication to requesters
- Complaint-driven improvement programs
- Building credibility for VoC systems

## Core Practices

- Capture feedback with enough structure to aggregate
- Prioritize with transparent criteria (reach, impact, effort, strategy fit)
- Decide deliberately: build, defer, or decline
- Communicate decisions back to contributors when possible
- Celebrate closed loops publicly (release notes, champions)
- Measure loop time and closure rate

## Principles

- Silence after feedback is a trust tax
- Declining requests respectfully is better than false hope
- Not all feedback should shape the roadmap — and that’s OK if explained
- CSMs need status visibility to set expectations

## Verification

- [ ] Feedback has a path to prioritization decisions
- [ ] Contributors sometimes receive outcomes communication
- [ ] Loop metrics are reviewed
