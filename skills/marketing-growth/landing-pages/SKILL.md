# Landing Pages

## Overview

A landing page has one job: convert a specific visitor intent into a specific action. Everything on the page should serve that job.

## When to Use

- Paid campaign destinations
- Product or feature launch pages
- Lead capture and waitlists
- Event or webinar registration

## Core Structure

1. Headline aligned to traffic source intent
2. Supporting subhead / value clarity
3. Social proof and credibility
4. Benefits and how it works
5. Objection handling
6. Primary CTA (repeated as needed)
7. Secondary trust elements (FAQ, guarantees, logos)

## Principles

- Message match between ad/source and page
- One primary goal per page
- Above-the-fold clarity matters enormously
- Speed and mobile quality are conversion factors
- Test hypotheses; don’t redesign blindly

## Verification

- [ ] Visitor intent and page promise match
- [ ] CTA is obvious and repeated appropriately
- [ ] Proof is visible before the ask when trust is low
