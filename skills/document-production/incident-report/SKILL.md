# Incident Report

## Overview

Incident reports create a reliable record for learning and accountability. They focus on facts, timeline, impact, and improvement — not blame.

## When to Use

- Production or operational incidents
- Security incidents
- Safety or compliance incidents
- Significant customer-impacting events

## Recommended Structure

1. Summary (what, when, impact)
2. Timeline of key events
3. Impact assessment
4. Detection and response actions
5. Root cause analysis
6. Contributing factors
7. Corrective and preventive actions
8. Lessons learned
9. Appendix (logs, graphs, evidence)

## Principles

- Stick to verifiable facts in the timeline
- Quantify impact where possible
- Separate immediate fixes from longer-term improvements
- Assign owners and due dates to action items
- Write so someone uninvolved can understand the event

## Verification

- [ ] Timeline is clear and factual
- [ ] Impact is described concretely
- [ ] Actions have owners
- [ ] Root cause analysis goes beyond the surface symptom
