# PPTX – Presentation Production

## Overview

Build presentation decks that communicate clearly in 5–10 seconds per slide. Favor restraint, strong hierarchy, and consistent design over flashy effects.

## When to Use

- User requests slides, a deck, presentation, pitch, or .pptx
- Content needs to be presented visually to an audience
- Executive summaries, project updates, proposals, or training materials

## Core Principles

- One main idea per slide
- Strong visual hierarchy (title → key point → supporting detail)
- Consistent alignment, spacing, and margins across the entire deck
- Limited color palette and font set
- Prefer solid fills and simple shapes over heavy gradients and shadows
- Readable from a distance (sufficient font sizes and contrast)
- Avoid walls of text — use short bullets, diagrams, or visuals

## Recommended Structure

1. Title slide
2. Agenda / agenda-lite (optional)
3. Problem or context
4. Key insights or solution
5. Supporting detail / evidence
6. Next steps or call to action
7. Appendix (if needed)

## Design Rules of Thumb

- Title: 28–40 pt
- Body: 18–24 pt
- Maximum ~6–8 lines of text per slide
- Generous whitespace
- Align elements to a grid
- Use icons sparingly and consistently

## Verification

- [ ] Deck tells a coherent story when clicked through quickly
- [ ] Every slide has a clear takeaway
- [ ] Layout is consistent from slide to slide
- [ ] Text is readable and high-contrast
- [ ] No overlapping elements or cut-off text
- [ ] File is saved as .pptx

## Common Pitfalls

- Too much text per slide
- Inconsistent alignment and spacing
- Decorative elements that compete with content
- Low-contrast text or icons
