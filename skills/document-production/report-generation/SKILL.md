# Report Generation

## Overview

A good report answers the reader’s key questions quickly, supports claims with evidence, and makes the next actions obvious.

## When to Use

- Status reports, project updates, incident reports
- Analytical or research findings
- Executive summaries and decision-support documents
- Any request for a formal written report

## Recommended Structure

1. **Title & metadata** (author, date, version, audience)
2. **Executive summary** (key points + recommendation in <1 page)
3. **Context / background**
4. **Findings or analysis** (with evidence)
5. **Implications / options**
6. **Recommendations or next steps**
7. **Appendix** (detailed data, methodology, references)

## Principles

- Lead with the answer, then support it
- Separate facts from interpretation
- Make recommendations specific and actionable
- Keep the main body concise; push detail to appendices
- Match tone and depth to the audience (executive vs. technical)

## Verification

- [ ] Executive summary can stand alone
- [ ] Claims are supported by data or sources
- [ ] Next steps are clear and owned where possible
- [ ] Structure is easy to navigate (headings, table of contents for longer reports)
