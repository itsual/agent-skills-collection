# One-Pager

## Overview

A one-pager forces prioritization. It is a high-signal summary designed to be read in 1–2 minutes.

## When to Use

- Project or initiative overviews
- Executive briefing sheets
- Proposal leave-behinds
- Status at-a-glance documents
- Idea or concept summaries

## Typical Sections

- Title and one-sentence description
- Problem / opportunity
- Proposed approach or current status
- Key benefits or metrics
- Timeline or milestones
- Ask / next step / decision needed
- Contact or owner

## Principles

- Ruthlessly cut anything that is not essential
- Use visual hierarchy and short blocks of text
- Prefer bullets and bold lead-ins over long paragraphs
- Make the “so what” and the “ask” impossible to miss
- Design so it still works when printed

## Verification

- [ ] Fits comfortably on one page
- [ ] Core message is clear within 30–60 seconds
- [ ] Action or decision required is explicit
