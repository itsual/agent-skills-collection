# Presentation Storytelling

## Overview

Slides support a story; they are not the story itself. Strong presentations have a clear narrative arc, a defined audience, and a memorable core message.

## When to Use

- Creating or refining pitch decks, strategy presentations, or project updates
- When a deck feels like a data dump rather than a coherent argument
- Preparing for an audience that needs to make a decision or change their mind

## Core Narrative Patterns

- **Situation → Complication → Question → Answer**
- **What is → What could be → How we get there**
- **Problem → Insight → Solution → Proof → Ask**

## Principles

- Define the one thing you want the audience to remember
- Open with context the audience already accepts, then introduce tension
- Make the recommendation or “ask” explicit
- Use evidence to support, not to overwhelm
- End with clear next steps or a decision request

## Practical Tips

- Write the story in sentence form before designing slides
- Cut any slide that does not advance the narrative
- Use the appendix for backup data instead of overloading the main flow
- Rehearse the spoken narrative — slides should support speaking, not replace it

## Verification

- [ ] A listener can state the core message after one pass
- [ ] Each section has a clear role in the overall argument
- [ ] The “ask” or conclusion is unmistakable
- [ ] Supporting detail does not bury the main point
