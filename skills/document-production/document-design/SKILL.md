# Document Design

## Overview

Good document design makes content easier to understand and more credible. It is not decoration — it is communication.

## When to Use

- Creating any formal document or presentation
- Reviewing a draft for visual and structural quality
- Establishing templates or design standards

## Core Principles

- **Hierarchy** — Make the most important information the most visible
- **Consistency** — Same elements should look and behave the same way throughout
- **Alignment** — Elements should line up to a clear grid or edge
- **Whitespace** — Give content room to breathe; avoid crowding
- **Contrast** — Ensure text and key elements stand out from the background
- **Restraint** — Limit fonts, colors, and styles

## Practical Rules

- Maximum 2–3 typefaces per document
- Consistent margin and column structure
- Clear distinction between title, section headings, body, and captions
- Tables and figures should be referenced and labeled
- Avoid centered body text for long passages
- Prefer left-aligned text for readability in most business documents

## Verification

- [ ] A reader can grasp the structure in a few seconds
- [ ] Visual hierarchy matches information importance
- [ ] No accidental misalignment or uneven gaps
- [ ] Document looks intentional rather than default-template
